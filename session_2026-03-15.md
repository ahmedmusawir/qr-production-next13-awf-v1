**Task:** Fix Docker build failure — `/api/admin/route` export error (exit code 1)

**Fix:** Added `export const dynamic = "force-dynamic"` to `/app/api/admin/route.ts`

### 10:30 — COMPLETE
**Task:** Fix middleware redirect loop on DO box (ERR_TOO_MANY_REDIRECTS)
**Root Cause 1:** `/login` not excluded from middleware matcher → redirect to /login triggered middleware → infinite loop
**Root Cause 2 (deeper):** `/login` doesn't even exist in this app. Home page is public. The auth redirect in middleware was wrong for this app entirely.
**Fix:** Removed the login redirect from `utils/supabase/middleware.ts` entirely. Middleware now only refreshes the Supabase session. Auth protection lives at page level.
**Files changed:** `utils/supabase/middleware.ts`, `middleware.ts`

### 11:00 — COMPLETE
**Task:** Fix missing users on superadmin portal in Docker production
**Root Cause:** `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` (and GHL equivalents) were NOT passed as Docker build args → browser bundle had `undefined` for both → Supabase browser client silently returned [] for all queries
**Fix:** Updated Dockerfile to declare all 6 NEXT_PUBLIC_* vars as ARGs. Updated both build scripts to source their respective env files (`set -a; source .env.production; set +a`) then override URL vars. Build scripts now auto-read all NEXT_PUBLIC_* from env file.
**Files changed:**
- `Dockerfile`
- `docker-build.sh`
- `docker-build-local.sh`
