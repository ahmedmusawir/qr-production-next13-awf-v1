## Session Progress

### COMPLETE — Fix Docker build failure (`/api/admin/route` exit code 1)
**Root Cause:** `SUPABASE_SERVICE_ROLE_KEY` is runtime-only. Next.js tried to statically render the route at build time inside Docker where the key was `undefined`. Locally, `.env.local` was loaded so it worked fine. Classic "works on my machine" — different env context.
**Fix:** `export const dynamic = "force-dynamic"` on `app/api/admin/route.ts`

### COMPLETE — Pre-emptive force-dynamic sweep
**Root Cause:** Same static render problem would have hit more routes. Also `app/layout.tsx` was async and calling `fetchAndGenerateTicketTypes()` + `fetchAndGenerateValidOrderList()` at render time — those made network calls to `cyberizewebdevelopment.com:4004` (unreachable during build) causing 10s timeouts per page.
**Fix:** Added `force-dynamic` to 7 routes + `app/layout.tsx`
- `app/api/ghl/events/route.ts`
- `app/api/ghl/price/route.ts`
- `app/api/ghl/orders/route.ts`
- `app/api/qrapp/timer/route.ts`
- `app/api/qrapp/sync-status/route.ts`
- `app/api/qrapp/orders/total-orders/route.ts`
- `app/layout.tsx`

### COMPLETE — Fix admin search term persisting across navigation
**Root Cause:** Zustand `searchTerm` survives soft navigation. Component remounts with `useState("")` (empty input, no X button) but store filter still active. Hard page refresh was the only fix because it wiped Zustand.
**Fix:** `setSearchTerm("")` in the `useEffect` cleanup of `AdminPortalPageContent` — resets on unmount so every visit starts clean.

### COMPLETE — Create local Docker scripts
Three new scripts for local Docker testing so they don't collide with prod:
- `docker-build-local.sh` — image `qr-app-local`, URLs `localhost:4004`, sources `.env.local`
- `server-start-local.sh` — container `qr-app-local`, `--env-file .env.local`
- `server-stop-local.sh` — stops/removes `qr-app-local`

### COMPLETE — Fix ERR_TOO_MANY_REDIRECTS on DO box
**Root Cause:** App has no `/login` page and the home page is public. But `utils/supabase/middleware.ts` was hardcoded to redirect any unauthenticated request to `/login`. Worked locally because Tony was already logged in (session cookie). On fresh DO box = no session = redirect to `/login` = middleware fires again = infinite loop.
**Fix:** Ripped out the redirect block entirely. Middleware now only refreshes the Supabase session and passes through. Auth protection lives at the page level where it belongs.

### COMPLETE — Fix missing users on superadmin portal in Docker
**Root Cause:** `NEXT_PUBLIC_*` vars (Supabase URL, anon key, GHL URL, GHL location ID) must be baked into the client-side JS bundle at build time. The Dockerfile only had `NEXT_PUBLIC_API_BASE_URL` and `NEXT_PUBLIC_SOCKET_URL` as build args. The other 4 were in `.env.production` (runtime only) — server-side code could see them, but the browser bundle had `undefined`. Supabase browser client initialized with `undefined` → silently returned `[]` for all queries.
**Fix:**
- `Dockerfile` — added all 6 `NEXT_PUBLIC_*` vars as `ARG`/`ENV`
- `docker-build.sh` — now does `set -a; source .env.production; set +a` to pull all `NEXT_PUBLIC_*` from the env file, then overrides the URL vars with hardcoded production values
- `docker-build-local.sh` — same pattern, sources `.env.local`

### COMPLETE — nginx + SSL (Phase 2)
Created `/etc/nginx/sites-available/qrtickets.cyberizewebdevelopment.com` on DO box.

Updated `docker-build.sh` URLs from `http://cyberizewebdevelopment.com:4004` → `https://qrtickets.cyberizewebdevelopment.com`.
Rebuilt and restarted. App is live with SSL.
